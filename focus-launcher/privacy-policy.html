プライバシーポリシー

最終更新日: 2026年3月28日

Focus Launcher（以下「本アプリ」）は、デジタルデトックスを目的としたAndroidランチャーアプリです。本プライバシーポリシーは、本アプリが収集・利用する情報について説明します。

1. 端末内で処理する情報

以下の情報は端末内でのみ処理・保存し、外部サーバーへ送信しません。

  • アプリ使用状況: どのアプリをいつ起動・使用したかの記録。タイムライン表示のためにのみ使用し、端末内のデータベースにのみ保存します。
  • Wi-Fi ネットワーク名（SSID）: ユーザーがWi-Fiトリガーのルールを設定する際に、接続中のネットワーク名を読み取ります。実際の位置情報（緯度・経度）は取得・記録しません。
  • ユーザー設定・ルール: ブロックルール、ホーム画面のアプリ設定などの設定情報。端末内にのみ保存します。

2. 外部サービスに送信する情報

本アプリはアプリの品質改善のために、以下の外部サービスを利用しています。

  • Firebase Crashlytics: アプリのクラッシュ情報（スタックトレース、端末モデル、OSバージョン等）を自動的に収集し、Googleのサーバーに送信します。個人を特定する情報は含まれません。
  • Firebase Analytics: アプリの使用状況に関する匿名の分析データ（オンボーディングの進捗状況、機能の利用頻度等）を収集します。個人を特定する情報は含まれません。

これらのサービスのプライバシーポリシーについては、Firebase のプライバシーとセキュリティをご確認ください。

本アプリが収集・送信する情報には、アプリの使用履歴（どのアプリをいつ使用したか）、ブロックルールの内容、Wi-Fi ネットワーク名、ジャーナルの内容は一切含まれません。

3. 利用する権限とその目的

  • アクセシビリティサービス（AccessibilityService）: 起動されたアプリのパッケージ名を検知し、ブロックルールに従って警告画面を表示するためにのみ使用します。キーロガー、パスワードの読み取り、画面内容の記録、ユーザーのタップ操作の追跡は一切行いません。
  • 使用状況へのアクセス（PACKAGE_USAGE_STATS）: アプリの使用履歴を取得し、タイムライン画面に表示するために使用します。
  • 他のアプリの上に重ねて表示（SYSTEM_ALERT_WINDOW）: ブロックルールが発動した際に、アプリの上に警告画面を表示するために使用します。
  • 位置情報（ACCESS_FINE_LOCATION）: AndroidがWi-FiネットワークのSSIDを読み取るために位置情報権限を要求します。本アプリは実際の位置情報（GPSやネットワーク経由の緯度・経度）を取得・保存・送信しません。
  • バッテリー最適化の無視: バックグラウンドでのアプリ監視サービスが停止しないようにするために使用します。
  • 起動完了の受信（RECEIVE_BOOT_COMPLETED）: 端末の再起動後にサービスを自動で再開するために使用します。
  • インターネット（INTERNET）: Firebase Crashlytics / Analytics のデータ送信に使用します。

4. データの保持

  • 端末内データ: アプリ使用状況、ブロックルール、ジャーナル等のデータは、端末内のデータベースに保存されます。アプリをアンインストールすると、端末内のすべてのデータは完全に削除されます。
  • Firebase経由のデータ: Firebase CrashlyticsおよびAnalyticsに送信されたデータは、Googleのデータ保持ポリシーに従って管理されます。詳細はFirebase のプライバシーとセキュリティをご確認ください。

5. データの削除

  • アプリ使用統計データ: 設定画面の「使用データを削除」から、記録されたアプリ使用履歴を手動で削除できます。
  • 全データの削除: アプリをアンインストールすることで、端末内に保存されたすべてのデータ（使用状況、ルール、ジャーナル、設定）が完全に削除されます。
  • Firebase データ: Firebase経由で送信された匿名データの削除については、Firebaseのプライバシーポリシーに準じます。

6. データの共有

本アプリは、上記のFirebaseサービスを通じてクラッシュレポートおよび匿名の分析データをGoogleに送信する以外に、収集した情報を第三者と共有・売却・提供することはありません。

7. プライバシーポリシーの変更

本ポリシーを変更する場合は、このページを更新します。定期的にご確認ください。

Privacy Policy

Last updated: March 28, 2026

Focus Launcher ("the App") is an Android launcher designed to help users with digital detox. This Privacy Policy explains what information the App collects and how it is used.

1. Information Processed Locally

The following information is processed and stored solely on your device and is never transmitted to external servers.

  • App Usage Data: Records of which apps were launched and when. Used solely to display the usage timeline, stored only in the local on-device database.
  • Wi-Fi Network Name (SSID): When you configure a Wi-Fi-based blocking rule, the App reads the name of the currently connected network. Actual location data (latitude/longitude) is never collected or stored.
  • User Settings and Rules: Blocking rules and home screen app configurations. Stored only on-device.

2. Information Sent to External Services

The App uses the following external services to improve quality and stability.

  • Firebase Crashlytics: Automatically collects crash information (stack traces, device model, OS version, etc.) and sends it to Google's servers. No personally identifiable information is included.
  • Firebase Analytics: Collects anonymous usage analytics (onboarding progress, feature usage frequency, etc.). No personally identifiable information is included.

For details on these services' privacy practices, please see Firebase Privacy and Security.

The App never collects or transmits your app usage history (which apps you used and when), blocking rule contents, Wi-Fi network names, or journal entries.

3. Permissions and Their Purpose

  • Accessibility Service: Used solely to detect the package name of the launched app and display a blocking screen when a rule is triggered. The App does not perform keylogging, read passwords, record screen content, or track touch input in any way.
  • Usage Access (PACKAGE_USAGE_STATS): Used to retrieve app usage history for display in the timeline screen.
  • Display Over Other Apps (SYSTEM_ALERT_WINDOW): Used to show a blocking overlay when a blocking rule is triggered.
  • Location (ACCESS_FINE_LOCATION): Android requires this permission to read Wi-Fi SSIDs. The App does not collect, store, or transmit actual geographic location (GPS or network-based coordinates).
  • Ignore Battery Optimizations: Used to prevent the background monitoring service from being killed by the system.
  • Receive Boot Completed: Used to restart the monitoring service automatically after device reboot.
  • Internet: Used to send data to Firebase Crashlytics and Analytics.

4. Data Retention

  • On-device data: App usage data, blocking rules, journal entries, and other data are stored in a local database on your device. Uninstalling the App permanently deletes all on-device data.
  • Firebase data: Data sent via Firebase Crashlytics and Analytics is managed according to Google's data retention policies. For details, see Firebase Privacy and Security.

5. Data Deletion

  • Usage data: You can manually delete all recorded app usage history from Settings → "Delete Usage Data".
  • All data: Uninstalling the App permanently deletes all data stored on your device (usage data, rules, journal entries, and settings).
  • Firebase data: For deletion of anonymous data sent via Firebase, please refer to Firebase's privacy policy.

6. Data Sharing

Other than sending crash reports and anonymous analytics data to Google through the Firebase services described above, the App does not share, sell, or provide any collected information to third parties.
